## 完成的工程项

1. **查询分解**：使用可审计的半导体中英文词表，不使用 LLM 生成改写。
2. **批量多路向量召回**：原问题和子查询一次嵌入请求，降低网络往返。
3. **概念覆盖精排**：每个子查询保留一个 Top-K 席位，再用整体得分补齐。
4. **严格型号防幻觉**：`QFAB-X99` 类含数字的具体型号必须精确命中才准入。
5. **标题感知切片和相邻证据**：Markdown 保留标题路径，相邻扩展严禁跨文档。
6. **切片扫描缓存**：混合检索缓存 Milvus 切片快照，默认 TTL 300 秒；
   入库、删除文档和删除集合时主动失效。
7. **端到端稳定性**：无证据 RAG 请求确定性拒答；越界引用服务端清理；
   复合问题逐项回答并保留英文专业术语。
8. **评测器增强**：空白归一化、金标来源召回、被引用文档、引用越界、
   禁止词、拒答和 P95 延迟统一输出。
9. **运行安全**：默认不记录完整提示词；资讯 API 未配置时不执行无效启动采集。
10. **存储缺陷修复**：修复 `get_chunks_by_filename()` 方法主体不可达问题。

## 仍然存在的边界

1. 20 题是工程回归集，不能代表完整业务准确率；应扩展到至少 80 题并保留隐藏测试集。
2. 中英文领域词表需版本化，避免随着题集不断堆叠规则。
3. NIST PDF 仍有双栏、页眉和表格结构丢失；标题感知目前主要惠及 Markdown。
4. 16 个正例中 1 个回答引用了替代 NIST 权威证据，没有引用唯一人工首选文档；
   评测单独报告该指标，但不将有效替代证据判为错误。
5. PyMilvus ORM API 已被上游标记弃用，后续需迁移到 `MilvusClient`。
