# 半导体产业研究助手 RAG 评测基线

评测日期：2026-07-15

## 1. 评测目标

本基线不只检查“模型能否回答”，而是分层评估：

1. 检索层：正样本能否命中证据，负样本能否被低分过滤。
2. 回答层：关键事实覆盖、引用合法性、无证据拒答、因果不确定性、生产动作安全边界与延迟。
3. 可复现性：在相同配置下重复运行，比较回答、引用和指标是否一致。

## 2. 环境与配置

- GPU：NVIDIA GeForce RTX 4060 Laptop GPU（8 GB）
- 生成模型：`industry-qwen3:4b`，基于 `qwen3:4b-instruct`
- Embedding：`bge-m3`，1024 维
- 检索：Milvus 向量检索 + 轻量词法检索
- 生成温度：0
- 模型上下文：8192 tokens
- 检索证据预算：6000 tokens
- 最大回答：1024 tokens

## 3. 评测集

| 方向 | 检索用例 | 端到端用例 |
|---|---:|---:|
| 芯片设计与 EDA/IP | 6 | 3 |
| 半导体材料与设备 | 6 | 3 |
| 晶圆制造与前道工艺 | 6 | 3 |
| 封装与测试 | 6 | 3 |
| 合计 | 24 | 12 |

问题类型包括事实查询、排查步骤、编号检索、知识边界、无证据问题、因果误判与生产安全边界。

## 4. 结果

### 4.1 检索层

- 总通过率：24/24
- 正样本命中率：21/21
- 负样本拒绝率：3/3
- 四个知识库均为 6/6

### 4.2 端到端回答层

| 运行 | 通过率 | 平均延迟 | P95 延迟 |
|---|---:|---:|---:|
| 基线运行 | 12/12 | 2.45 s | 5.49 s |
| 重复运行 | 12/12 | 2.42 s | 4.47 s |

两次运行中，12/12 用例的回答文本、检索文档数、引用编号、事实覆盖率与通过状态完全一致。

## 5. 评测驱动的优化

初始回答评测为 11/12。失败用例询问是否可以自动调整 ETCH-07 真空阀；检索结果排名第一的片段已包含“不授权自动调整设备参数”、OCAP 和责任工程师要求，但生成模型错误回答“知识库未提供”。

根因不是检索失败，而是生成提示词未明确处理“执行动作/权限”问题。增加通用安全规则后，单条复测和完整回归均通过。

另外，将本地模型温度从 0.2 调整为 0，以提高 RAG 回答和回归测试的可复现性。

## 6. 复现命令

```bash
cd backend/app

PYTHONPATH=. python scripts/evaluate_rag_retrieval.py \
  --cases ../../sample-data/rag_evaluation_cases.json \
  --top-k 3 \
  --output ../../reports/rag_retrieval_baseline.json

PYTHONPATH=. python scripts/evaluate_rag_answers.py \
  --cases ../../sample-data/rag_answer_evaluation_cases.json \
  --model-mode auto \
  --max-latency 20 \
  --output ../../reports/rag_answer_baseline.json
```

## 7. 局限与下一步

1. 当前每个方向只有一份合成演示文档，还不能代表真实企业知识库。
2. 端到端评分为可解释的确定性规则，尚未引入人工双人标注或 LLM-as-a-judge 对比。
3. 当前是单用户串行测试，尚未评估并发量、首 token 延迟与长文档压力。
4. 下一轮应优先增加真实公开资料，并建立“事实、时间、版本、来源权威性”的多文档冲突评测。
