# 论断—引用—证据对齐评测

日期：2026-07-17

## 为什么“有引用”还不够

一个回答即使包含 `[[1]]`，仍可能存在三类错误：

1. 编号越界或指向不存在的参考内容；
2. 整段只在末尾有一个引用，其他事实句无法追溯；
3. 编号合法，但对应切片并不支持该句使用的概念或参数。

因此评测器现在建立如下可审计链路：

```text
必需术语组/同义词
        ↓
回答中包含该概念的句子
        ↓
该句子自带的 [[n]] 引用
        ↓
第 n 个真实检索切片
        ↓
切片是否包含同一术语组中的概念/同义表达
```

这是确定性 lexical support 闸门，不是语义蕴含模型。它优点是可重现、无评委模型
偏差；局限是可能把“语义支持但未使用标注同义词”判为失败。后续应将它与
cross-encoder/NLI 评委并行使用，而不是用模型评委取代确定性闸门。

## Regression 严格重跑

复现（需启动后端与本地模型）：


这组结果取代“回答只要有一个合法引用就通过”的宽松口径。历史 20/20 仍保留，
但不再用作引用完整性证明。

## 失败归因

| 用例 | 支持覆盖 | 门槛 | 主要问题 |
| --- | ---: | ---: | --- |
| `materials-neon-007` | 33.3% | 67% | 地理集中/供应中断句未对齐直接切片 |
| `materials-metrology-009` | 50% | 75% | 参考材料缺失，污染物句引用不支持对应概念 |
| `process-inline-011` | 50% | 75% | 过程控制/质量保证存在未引用或错位引用 |
| `process-digital-twin-013` | 25% | 75% | 数字孪生定义和真实/物理世界句没有直接对齐 |

在提示词中新增“每个事实句必须自带直接支持引用”后，这4题仍为 0/4。
这证明继续堆提示词不是可靠解法。

## 工程结论

1. 生成层应输出结构化 `claims[]`，每个 claim 带 `citation_ids[]` 和不确定性；
2. 服务端先校验编号、逐论断支持和引用完整性，再渲染为自然语言；
3. 确定性术语组闸门与 NLI/cross-encoder 必须同时报告，并保留分歧样本人工复核；
4. 未通过支持审核的 claim 必须删除、改写为不确定表达，或触发补充检索；
5. 不得通过把门槛从 75% 下调来恢复“20/20”。
