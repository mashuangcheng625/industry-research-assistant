# blind-v2 首次评测审计

## 结论

blind-v2 已在代码和索引冻结后完成首次 25 题评测，但本次运行暴露出题集的知识库映射与
来源标注缺陷。因此，这份题集从本次运行起视为**已解盲的诊断集**，结果不能作为模型准确率
或泛化能力的对外结论，也不得据此修改检索词表、问题或私有答案键。

## 数据质量缺陷

| 缺陷 | 影响 |
| --- | --- |
| 4 道 RISC-V 架构正例被分配到“晶圆制造与前道工艺”知识库 | 金标 RISC-V 文档只按已审核来源路由进入芯片设计库，这 4 题在指定集合内不可能命中金标 |
| 封装题把 `riscv-unprivileged-20260120.md` 标成 UCIe 2.0 来源 | 原始规范文档中不存在 UCIe 内容，属于来源错引 |
| `nist-materials-equipment-vision-2023.md` 被封装题指定为金标 | 已审核来源路由未将该文档放入封装库，指定集合内无法召回 |
| 部分问题把跨领域材料同时绑定到单一知识库 | Top 5 被同域但非金标文档占据，来源 Recall@5 被路由契约而非向量质量限制 |
| 评分只做关键词子串覆盖 | 同义表达可能被误判，当前百分比不能等同于语义正确率 |

这些问题是评测资产缺陷，不应通过扩大 Top K、读取私有答案后添加查询规则或修改题目来掩盖。

## 后续验收规则

1. blind-v2 保留原始问题与答案，不再用于调参，只用于复现本次诊断。
2. 通用检索改进只能在 development 上开发，并用 regression 防回退。
3. 新一轮结论使用独立维护者在代码冻结后制作的 blind-v3；制作前必须校验每个金标文档
   已进入题目指定知识库，且问题所述章节和事实真实存在。
4. blind-v3 报告分别给出来源 Recall@K、回答覆盖、引用支持、拒答和延迟，不再只报一个总分。
5. 新题集首次运行后立即封存，并以相对 development/regression 基线的变化报告结果。
