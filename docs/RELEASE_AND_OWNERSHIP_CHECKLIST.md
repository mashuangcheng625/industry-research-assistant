# 发布与权属检查清单

本清单用于求职展示、向 GitHub 公开或向第三方分发前的人工确认。删除文件头声明不会改变代码、数据或文档的实际权属。

## 当前发布状态

- 仓库默认保持私有；
- 明确不添加开源许可证，仓库按保留全部权利的私有面试展示项目管理；
- 未完成下列人工确认前，不将仓库描述为可自由复制、修改或再分发的开源项目；
- NIST、RISC-V、OpenROAD、IEEE 等外部资料继续遵循各自来源条款，来源注册表中的许可字段不得删除。

## 必须由项目负责人确认

- [ ] 确认所有应用代码均为本人原创，或已取得雇主/委托方的书面展示与发布许可；
- [ ] 列出曾在公司设备、公司时间或公司项目中产生的文件，并确认是否允许迁移到个人仓库；
- [ ] 确认界面素材、截图、图标、字体和样例数据的再分发权利；
- [ ] 确认报告中的模型、API、数据源和运行结果不包含公司秘密或个人信息；
- [ ] 为每一份外部语料保留来源 URL、许可策略、哈希和审核结论；
- [x] 明确采用保留全部权利的私有展示方式，不添加 LICENSE；
- [ ] 在公开前用干净 clone 执行 `make check`，并复核 Git 历史中不存在密钥、私有答案或受限文件。

## 建议保留的证据

- 书面授权或劳动/委托协议中与作品归属相关的条款；
- 个人贡献记录和可审计提交历史；
- 第三方依赖清单及其许可证扫描结果；
- 数据来源审核记录与内容哈希；
- 对外发布版本对应的 commit SHA 和 CI 链接。
