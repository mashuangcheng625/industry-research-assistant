# 先进封装与测试良率演示语料

> 数据声明：本文档是为本地 RAG 功能测试编写的合成案例，不代表任何真实封装产线或产品规格。

## 1. 从晶圆测试到成品测试的追溯链

Chiplet 和 2.5D/3D 封装同时涉及多颗裸片、中介层或基板、微凸点/键合结构与多道测试。追溯时需保留 die ID、原始 wafer/lot、晶圆测试 bin、组装设备和时间、基板与材料批次、工艺 Recipe、AOI/X-ray/SAM 检查结果、烧机和成品测试 bin。

“已通过晶圆测试”不等于组装后一定良好。封装会引入互连开短路、翅曲、分层、热机械应力和散热路径等新风险，因此需将前段测试、封装过程与最终失效 bin 连接起来。

## 2. 合成案例：2.5D 模组高速链路开路 bin 上升

演示产品 `PKG-DEMO-X2` 的成品测试中，高速链路开路 bin 从基线 0.7% 升至 3.1%。异常集中于组装线 Assembly-B 的两个生产日，但来自多个晶圆测试 lot。对应时间段的凸点 AOI 显示边缘区高度离散增大，X-ray 中部分失效件在相近区域有可疑互连特征。同时该时段更换了一批基板，但贴装机校准和回流曲线也有维护记录。

排查应先确认测试程序、探针/测试座和复测结果，排除测试假失效；然后按 unit ID 联接晶圆测试、AOI、X-ray、组装设备和材料批次；再对失效件做位置分布和解剖分析；最后用受控 split 区分基板批次、贴装校准和回流曲线等候选因素。现有数据只能说明时间与空间关联，不足以单独确认基板或设备为根因。

## 3. 检索回答要求

本案例支持的答案应包含：先排除测试系统假失效；再建立 die-to-package-to-test 追溯；关联 AOI、X-ray、组装线、基板批次和工艺维护记录；使用解剖与 split 验证候选根因。不应把单一关联项直接写成确定根因。
