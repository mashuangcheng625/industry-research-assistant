# 半导体工艺异常分析演示语料

> 数据声明：本文档是为本地 RAG 功能测试编写的合成案例，不代表任何真实晶圆厂、设备或工艺规范，不能用于实际生产决策。

## 1. 数据源与用途

工艺异常分析通常需要关联三组数据。第一组是设备与工艺时序数据，包括 FDC 传感器曲线、设备报警、腔体压力、射频功率、气体流量、温度以及 Recipe 版本。第二组是质量与检测数据，包括 SPC 控制图、CD 测量、膜厚、缺陷图、晶圆 Map 和电性测试结果。第三组是生产上下文与工程知识，包括 MES 批次履历、晶圆与设备加工轨迹、材料批次、设备保养记录、OCAP、8D 报告和历史失效分析结论。

只有把三组数据按 lot、wafer、equipment、chamber、recipe 和时间窗口对齐，才能避免把相关性误判为根因。例如，良率下降可能与腔体压力漂移同时出现，但还需检查 Recipe 是否变更、量测系统是否稳定，以及异常批次是否集中在同一材料批次。

## 2. 合成案例：ETCH-07 腔体压力异常

演示设备 ETCH-07 的 Chamber-B 出现腔体压力波动告警。合成规则编号为 `DEMO-OCAP-ETCH-01`。规则要求首先暂停受影响批次继续流转，并保存异常发生前后各 30 分钟的 FDC 曲线。随后关联以下信息：腔体压力、真空阀位置、气体流量、射频功率、Recipe 版本、最近一次 PM 时间、耗材更换记录，以及同批次晶圆的 CD 和缺陷检测结果。

排查顺序如下：先确认压力传感器与量测时间戳是否可信；再比较 Chamber-A 与 Chamber-B 在相同 Recipe 下的曲线；随后检查真空阀响应、气路状态和最近维护变更；最后查看 CD、刻蚀深度与缺陷分布是否同步偏移。若只有传感器值异常而产品量测和其他信号稳定，应优先排查传感器或数据采集链路，而不能直接认定工艺失控。

## 3. 检索回答要求

回答异常处置问题时，应区分“检索到的事实”“基于事实的推断”和“仍需验证的假设”。事实必须能够追溯到文档片段；推断应说明依据；假设不能写成确定结论。涉及真实生产处置时，必须以企业批准的 OCAP、EHS 要求和责任工程师判断为准。

对于“ETCH-07 腔体压力异常时应关联哪些数据”的问题，本文档支持的答案包括：FDC 压力曲线、真空阀位置、气体流量、射频功率、Recipe 版本、PM 与耗材记录，以及 CD 和缺陷检测结果。本文档不提供真实阈值，也不授权自动调整设备参数。
